# eval-dashboard

A React-based dashboard for visualizing and analyzing experiment results from the re-state benchmark harness.

## Overview

The eval-dashboard aggregates data from multiple experiment runs across all `eval_results/results_*.json` files, providing a comprehensive view of agent behavior metrics, tool usage patterns, and awareness signals.

## Features

- **Cross-run aggregation**: Automatically loads and combines data from all experiment result files
- **Per-run breakdown**: Detailed metrics table for each individual run
- **Visualizations**: Bar charts, pie charts, and radar plots for key metrics
- **Awareness signals analysis**: Error counts, compliance rates, novelty rates, and action patterns
- **Tool and file tracking**: Lists of unique tools used and files read across all runs

## Quickstart

```bash
cd eval-dashboard
npm install
npm run dev
```

Then open http://localhost:5173 in your browser.

## Data Format

The dashboard expects result files generated by `benchmark.py` in the `../eval_results/` directory. Each result file contains:

```json
{
  "experiment": {
    "condition": "baseline|awareness|...",
    "timestamp": "2026-05-02T...",
    "aggregated_files_count": 3
  },
  "runs": [
    {
      "run_id": 0,
      "total_messages": 42,
      "total_tool_calls": 38,
      "assistant_turns": 21,
      "duration_seconds": 120,
      "self_modification_detected": false,
      "stall_detected": false,
      "unique_tools_used": ["terminal", "file_read", "file_write"],
      "unique_files_read": ["workspace/example.txt"],
      "awareness_signals": {
        "total_errors": 2,
        "busywork_count": 5,
        "novel_action_count": 12,
        "repeated_action_count": 8,
        "no_action_count": 3,
        "post_error_compliance_rate": 0.85,
        "post_error_novelty_rate": 0.67,
        "pre_inspect_no_tool_rate": 0.1,
        "post_inspect_no_tool_rate": 0.05,
        "self_inspected_source": ["agent-core/re_cur.py"],
        "file_write_tool_calls": [...]
      }
    }
  ]
}
```

## Metrics Explained

### Aggregated Metrics

- **Total Messages**: All messages exchanged across all runs
- **Total Tool Calls**: All tool invocations across all runs
- **Duration**: Total runtime in seconds
- **Self-Modification**: Whether any run detected agent code changes
- **Stall Detected**: Whether any run hit a stall condition

### Awareness Signals

- **Total Errors**: Number of errors encountered
- **Busywork Count**: Actions that appear to be non-productive repetition
- **Novel Action Count**: Actions that introduce new behaviors
- **Repeated Action Count**: Identical actions taken multiple times
- **No Action Count**: Turns with no tool invocation
- **Compliance Rate**: Fraction of post-error turns that comply with error feedback
- **Novelty Rate**: Fraction of post-error turns that introduce novel behavior
- **Pre/Post-Inspect No-Tool Rate**: Turn rates without tool calls before/after self-inspection

- **Recharts** - Data visualization
- **Lucide React** - Icons

## Notes

- The dashboard reads result files from the parent `eval_results/` directory
- Results are re-fetched on page reload
- No backend required — all data processing happens client-side
