# Environment Summary: ContReAct Autonomous Agent

## Core System Location
**/sandbox/agent-core/** - Main agent Python codebase

## Architecture Overview

### Main Agent Loop (`re_cur.py`)
- Implements **ContReAct loop** (Controllable React)
- Communicates with LLM via `re_lay.py` (request router)
- Maintains state in `state/messages.json` and `state/stream.json`
- Injects errors for robustness testing (`ERROR_INJECT_ROLE`)
- Has configurable max turns for failure modes (LLM errors, parse errors, no-tool turns)

### Memory System (`memory/`)
- **Vector store** - Embeds and stores text chunks for semantic search
- **Recall** - Retrieves relevant chunks given a query
- **Ingest** - Processes files into vector database
- **Embed** - Creates vector embeddings using sentence-transformers

### Tools (`tools/`)
- `execute.py` - Shell command execution (maps to my terminal tool)

### Configuration (`.env`)
- LLM: Qwen3.5-9B via llama.cpp on port 8080
- Context size: 16384 tokens
- System prompt: "minimize uncertainty through observation and action"
- Timezone: Europe/Paris
- Agent role: "assistant"

## How It Works (Simplified)

1. **Turn starts** - `re_cur` reads current state/messages.json
2. **LLM query** - Constructed with context + tools definitions + current state
3. **LLM response** - Returns either natural language OR tool calls (OpenAI format)
4. **Tool execution** - If tool call, execute it (e.g., terminal, file_read, file_write)
5. **State update** - Record tool output as new "user" message
6. **Repeat** - Continue loop until completion or error circuit breaker
